Agent Protocol — SOUL.md

5 Mandatory Rules
0
Read first, always
On session start: read SESSION-STATE.md then RECENT_CONTEXT.md before anything else. Instant re-orientation after crashes.
→ rosepuppy SESSION-STATE pattern
1
Write-ahead rule
User gives a concrete fact, correction, or decision → update SESSION-STATE.md before responding. Never after.
→ rosepuppy write-ahead log
2
ADD / UPDATE / DELETE / NOOP
Before writing to semantic/ — search first, then: add new, supplement existing, remove contradicted, or skip if already there.
→ Mem0 two-phase write pipeline
3
Tier routing
Facts → semantic/   Steps → procedural/   Events → episodic/YYYY-MM-DD.md
→ memory-manager three-tier
4
Heartbeat capture
Every ~10 turns: capture facts, refresh RECENT_CONTEXT.md, update SESSION-STATE if task shifted. No user signal needed — session is assumed always open.
→ rosepuppy heartbeat · Mem0 rolling summary
