Protocol Layer — 5 Rules (SOUL.md)

Protocol

5 Mandatory Rules

0
Session Start: Read First
Always read SESSION-STATE.md + RECENT_CONTEXT.md before anything else. Instant re-orientation after crashes or restarts.
→ rosepuppy SESSION-STATE pattern
1
Write-Ahead Rule
When user gives a concrete fact, correction, decision, or preference — write to memory before responding. Not after. Not later.
→ rosepuppy write-ahead log
2
ADD / UPDATE / DELETE / NOOP
Before writing to semantic/ — search first. Then: ADDnothing exists · UPDATEsupplements · DELETEcontradicts · NOOPalready there
→ Mem0 two-phase pipeline
3
Tier Routing
Facts → semantic/   Steps → procedural/   Events → episodic/YYYY-MM-DD.md
→ memory-manager three-tier structure
4
Heartbeat Capture (every ~10 turns)
Capture new facts, update RECENT_CONTEXT.md, refresh SESSION-STATE.md if task shifted. Never waits for "goodbye" — session is assumed always open.
→ rosepuppy heartbeat · Mem0 rolling summary
